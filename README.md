# DataForge30 – 30 Days, 30 Real-World Data Projects

## Overview
**DataForge30** is a **30-day coding challenge** where I build **one real-world data project per day**.  
The goal is to practice and showcase **data engineering, Python, SQL, log analysis, automation, and NLP skills** in a structured, portfolio-ready format.

Each project includes:
- Clean, structured code
- Input and output files
- A mini README explaining objectives, tools, and learnings

---

## Challenge Objectives
- Improve **data engineering and analytics skills**
- Learn **real-world workflows**:
  - Data cleaning & transformation
  - Log analysis & KPI extraction
  - Automation & reporting
  - Security monitoring
  - NLP and workflow automation
- Create **reusable, professional-quality projects**
- Build a strong **GitHub portfolio**

---

## Technologies & Tools Used
- Python (Pandas, CSV/JSON handling)
- SQL (for database-related projects)
- Google Colab / Jupyter Notebook
- n8n for workflow automation (optional)
- Git & GitHub for version control

---

## Project List (Days 1–6 Completed)

| Day | Project Name | Description |
|-----|--------------|-------------|
| 1 | CSV Data Cleaner | Clean and preprocess CSV datasets |
| 2 | Server Log KPI Analyzer | Analyze server logs for traffic and error KPIs |
| 3 | Top Endpoint Usage Analyzer | Identify most-used endpoints in logs |
| 4 | Error Log Alert Generator | Extract error requests (4xx/5xx) and generate alerts |
| 5 | Suspicious IP Activity Detector | Detect suspicious IPs based on request frequency |
| 6 | Daily Data Report Generator | Automate daily summary reports from datasets |

> ✅ Future days will continue adding **new real-world projects** up to Day 30.
